===== 浅利君解釈 ====
ExcelをgetSubjectDataに入れる
get~は関数
NW_trunkflexionを開く
plot_configも開く
〇plot_configの説明
・関節角度の上部に歩行周期の画像を添付
・縦軸のプロット範囲を自動で設定
・横軸をGaitで100%にする
・背景を歩行フェーズごとに着色（健常歩行の場合のもの）
・歩行フェーズの区切りごとに何の歩行フェーズかの説明
・下肢関節角度，重心上昇・横・前後，骨盤動揺など
・~は今何の関節を見ているかわかるようにしている（あまり気にしなくてOK）
・plot_config('見たい関節'); ※'見たい関節'を適切に入力しないとエラーが出る
※strdist.mはこの時のエラーメッセージを表示させるためのファイル
重心上昇を各歩行条件で比較したプログラム
・AS_CoM_z.m：通常歩行とアシスト歩行の比較
・DR_CoM_z.m：通常歩行とハンドリングの比較
・ハンドリングとアシストの比較←自分でコード書く
CoMのコードは，各歩行周期で重心最下点のデータを取得している．そのためのコードがある．
→他の関節データを比較したい場合はその点に注意
Xsensのデータは単位が[m]．重心上昇は[cm]にするためにデータを100倍している
データが増えるごとに，getSubjectDataのpathを変更していく
★実行するプログラムは，
"NW_trunkflexion.m"
"DR_CoM_z.m"
★実行手順
getSubjectDataのpathを変更していく
↓
プログラム実行

======= 阿部 ========
〇strdist.mはgoogleから検索してダウンロード
検索ワード：Matlab strdist file exchange
vs codeみたいにエラーの原因がスペルミスのときにDid you mean "～"?としたいがために使用


〇plot_withlogシリーズ
・INOMER様から頂いたログデータを関節角度変化に反映させるプログラム

・plot_withlog_angle
	データの種類　　
		関節角度：計測区間５ｍの平均値
		ログ：アシストタイミング開始終了の平均値
・plot_withlog_WHOLEdata
　小竹先生からデータの平均値よりタイミングのずれなどを見たいから計測全体で導出したらと
	データの種類
		関節角度：１１ｍ全体
		ログ：１１ｍ全体

ログデータの取得フレームがXSensと違うためタイムスタンプから導出していただいてる時刻でデータを合わせている
XSens：60Hz　　ログ：約50Hz

〇plot_withlog_WHOLEdata_each
　各歩行周期ごとにどうなっているか見たかったため重ねる形にしてみた

〇plot_comparison_ExperimentProcedure
　各手順ごとの関節角度変化の推移がわかりやすいような図を導出　
　ロボメックの第３章で使用

〇plot_CoM_FFT
・重心加速度の情報についてFFTを行って分析しようとした

〇plot_CoM
・各患者の重心の高さ変化について導出
・手順ごとによる違いも導出可
・通常歩行だけのものは、07clinical-analysisで使用

〇HipEx_NW
・股間節の関節角度変化を通常歩行時各患者の2試行分（往復路）の平均値と標準偏差範囲を導出するためのプログラム
・07clinical-analysisのグラフに該当
・・plot_CoMのコードを使用している。本当は、関節ごとに違うプログラムにしようと思ったが、面倒であったため、コードを変更することで、つま先の高さ（遊脚期最下点はトゥクリアランス）や膝関節の角度変化も導出している

〇NormalRange_ToeClearance
・患者のつま先の高さ変化を分析するときに健常者との比較が必要だと考えて使用
